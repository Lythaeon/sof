# Maintain SOF

This track is for contributors and maintainers working inside the repository.

Read this section if you are:

- changing runtime behavior
- modifying crate boundaries or architecture
- updating tests, fuzz targets, or release workflow
- documenting repository policy or operational posture changes

- workspace structure and slice ownership
- architecture rules and decision records
- contributor quality gates
- testing, fuzzing, and ADR discipline
